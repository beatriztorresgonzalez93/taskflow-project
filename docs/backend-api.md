# Backend API (Axios, Postman, Sentry y Swagger)

## Axios
**Axios** es una librería HTTP que simplifica requests y manejo de errores/JSON. En este proyecto el frontend usa `fetch`, pero se usa como referencia para entender alternativas.

## Postman
**Postman** permite crear colecciones de requests y validar respuestas (incluyendo casos de error) antes de consumir la API desde el frontend.

## Sentry
**Sentry** centraliza la observabilidad de errores en frontend y backend, útil para detectar fallos 500 y errores no controlados.

## Swagger (OpenAPI)
**Swagger/OpenAPI** describe la API mediante un contrato formal (endpoints y esquemas). Se usa para documentación navegable y para automatizar pruebas/cliente en proyectos reales.
